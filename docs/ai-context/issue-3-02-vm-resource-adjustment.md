# Issue #3-02: K8s 클러스터 VM 자원 수정을 위한 셧다운 자동화

## 📅 날짜: 2025-12-30
## 👤 참여자: 임성현, Gemini

## 1. 이슈 개요
- **목표**: K8s 워커 노드 및 마스터 노드(VM 300~305)의 vCPU/RAM 사양 변경을 위해 Proxmox API를 이용한 안전한 셧다운 수행.
- **결과**: Bastion 서버에서 API 토큰을 활용해 6개 VM에 대한 셧다운 명령 전송 완료.

## 2. 주요 성과 및 대화 요약
- **API 자동화**: 개별 VM 접속 없이 API를 통해 원격으로 클러스터 전체 노드 제어.
- **대화 요약**: 빠른 인프라 구성을 위해 직접 실습 대신 즉시 사용 가능한 스크립트 기반으로 작업 진행.

## 3. 기술적 의사결정 및 트러블슈팅
- **기술적 선택**: Proxmox API의 `status/shutdown` 엔드포인트 활용. 안전한 종료를 위해 `stop` 대신 `shutdown`을 선택하여 OS 레벨의 데이터 무결성 보장.
- **트러블슈팅**: 각 VM ID와 물리 호스트(server-01~06) 매핑 정보를 확인하여 정확한 API 호출 경로 생성.

## 4. 결론 및 향후 계획
- **결론**: 모든 노드 셧다운 후 Proxmox UI에서 하드웨어 스펙(CPU/RAM) 수정 대기 중.
- **차기 과제**: 자원 수정 완료 후 노드 기동, 앤서블 자동화 파일 생성 및 테스트, K8s 노드 레이블링(`workload-type`) 작업 착수.
